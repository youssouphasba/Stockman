# Guide - Administration

## 1. Role du module

Le module Administration sert au pilotage global de la plateforme : supervision, abonnements, utilisateurs, boutiques, securite, support et communications internes.

Profils concernes : super-administrateurs uniquement.

## 2. Acces

Barre laterale -> **Administration**.

## 3. Sections principales

Le backoffice peut regrouper des vues comme :

- vue d'ensemble ;
- abonnements ;
- demos ;
- utilisateurs ;
- boutiques ;
- produits et catalogue ;
- litiges ;
- securite ;
- support ;
- messages de diffusion.

## 4. Points utiles a retenir

### Catalogue admin

- l'import CSV admin peut etre traite par lots ;
- la progression d'import peut etre conservee ;
- une reprise d'import evite de recommencer un lot deja traite.

### Abonnements

Le backoffice permet de suivre :

- comptes payants ;
- trials ;
- statuts a risque ;
- historique et providers ;
- certains ajustements comme une grace ou un mode lecture seule selon les outils disponibles.

### Utilisateurs et boutiques

Les vues d'administration permettent de surveiller :

- comptes actifs ;
- plans ;
- pays ;
- boutiques rattachees ;
- certains evenements de blocage ou de moderation.

### Securite

On y retrouve les evenements de securite, les verifications et certaines sessions actives.

## 5. Questions frequentes

| Question | Reponse |
|---|---|
| Qui peut ouvrir ce module ? | Les super-administrateurs uniquement. |
| Le backoffice sert-il a gerer les abonnements ? | Oui, c'est une des fonctions principales du module. |
| Le suivi catalogue est-il aussi dans l'administration ? | Oui, notamment pour les imports, la revue et certains outils de pilotage global. |
