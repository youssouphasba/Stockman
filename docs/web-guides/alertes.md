# Guide - Alertes

## 1. Role du module

L'ecran Alertes regroupe les alertes in-app utiles a l'utilisateur : surtout le stock, puis certains rappels CRM, finance ou operations selon les reglages et le contexte.

## 2. Acces

- Depuis le module **Alertes**
- Ou via l'entree correspondante dans l'application quand une notification compatible ouvre directement cet ecran

## 3. Organisation actuelle

Les reglages et les rappels lies aux alertes doivent etre lus ensemble :

- les alertes in-app apparaissent en haut de l'ecran ;
- les rappels et blocs complementaires restent dans la meme zone fonctionnelle ;
- le but est d'eviter deux centres de reglage separes qui se contredisent.

## 4. Comportement attendu

- les alertes in-app privilegient les sujets vraiment utiles ;
- les alertes de stock sont prioritaires ;
- les alertes tres proches ou repetitives sont mieux regroupees ;
- les elements purement normaux ou peu actionnables ne doivent pas polluer la liste.

## 5. Canaux

Selon les regles et les droits, une alerte peut passer par :

- in-app ;
- push ;
- email.

Les destinataires doivent rester coherents avec les personnes autorisees a voir ou modifier ces rappels.

## 6. Regles d'alerte

Les reglages d'alerte servent a definir :

- les familles d'alertes utiles ;
- les canaux ;
- les destinataires ;
- le niveau de gravite minimal ;
- les rappels stock, CRM et finance selon les droits.

Les utilisateurs qui n'ont pas le bon plan ou les bonnes permissions ne doivent pas voir des regles reservees a un niveau superieur.

## 7. Questions frequentes

| Question | Reponse |
|---|---|
| Pourquoi je vois moins d'alertes qu'avant ? | Les alertes tres proches ou peu utiles sont mieux filtrees et regroupees. |
| Pourquoi l'alerte ouvre directement l'ecran Alertes ? | Certaines notifications compatibles redirigent maintenant vers l'ecran le plus utile pour traiter l'information. |
| Qui peut modifier les rappels ? | Seulement les personnes autorisees par le role, les permissions et le plan actif. |
