# Guide - CRM

## 1. Role du module

Le CRM centralise les fiches clients, la fidelite, les dettes, les promotions, les campagnes et les analyses client.

Profils concernes : utilisateurs ayant la permission `crm`.

## 2. Acces

Barre laterale -> **CRM**.

## 3. Ce que l'ecran permet

- creer et modifier des clients ;
- suivre les achats et l'encours de dette ;
- gerer les promotions ;
- lancer des campagnes selon les outils disponibles ;
- consulter des indicateurs et segments client ;
- exporter certaines donnees.

## 4. Regles a retenir

- les entrees du menu mobile doivent suivre le vrai code metier ;
- si une action est autorisee dans l'ecran, elle ne doit pas etre masquee a tort dans le menu ;
- les elements reserves a un niveau superieur doivent etre masques, pas simplement verrouilles visuellement.

## 5. Promotions et campagnes

Le CRM peut proposer :

- la creation de promotions ;
- le lancement de campagnes ;
- des rappels ou actions marketing lies a la base client.

La visibilite doit rester coherente avec les droits reels disponibles dans l'ecran.

## 6. Dettes et historique client

Depuis la fiche d'un client, on peut :

- consulter ses informations ;
- voir son historique d'achats ;
- suivre sa dette ;
- enregistrer ou annuler certains mouvements selon les droits accordes.

## 7. Questions frequentes

| Question | Reponse |
|---|---|
| Pourquoi un utilisateur ne voit pas une action CRM ? | Cela depend de ses permissions reelles et du plan actif. |
| Les promotions et campagnes sont-elles au meme endroit ? | Elles restent dans le CRM, mais leur visibilite doit suivre le comportement reel de l'ecran. |
| Le CRM est-il lie a la caisse ? | Oui, notamment pour la selection du client, l'historique d'achats et certains rappels. |
